# 🏗️ Architecture Notes — Совет колонии

_Брейншторм 19.06.2026. Ответы от 4 агентов по проекту API Hub._

---

## 🐍 Мангуст (аналитик-интегратор)

### 1. Архитектура
Микросервисы, но не классические — event-driven с легковесными воркерами. Монолит не потянет 20+ провайдеров с разными latency SLA. Serverless — дорого при постоянной нагрузке.

### 2. Стек
- Гейтвей: **Go** (gin или echo) — низкая latency, простой деплой
- Маршрутизация/оркестрация: **Go** или **Rust** (если нужен zero-cost abstraction)
- Периферийные сервисы (парсинг ответов, rate limiting): **Python/FastAPI** для быстрой разработки
- Очередь: **Redis Streams** или **NATS** для буферизации запросов к медленным провайдерам

### 3. Маршрутизация
Три слоя:
- L1 → выбор провайдера (по типу запроса)
- L2 → round-robin + weighted priority (по reliability score)
- L3 → circuit breaker на каждого провайдера

Fallback цепочка: primary provider → secondary → degraded response (кэш). Healthcheck каждые 15s. Если все упали — вернуть ответ из кэша с пометкой stale.

### 4. Хранение ключей
Не расширяй lab-vault. Сделай отдельный сервис-обёртку над **HashiCorp Vault** — там есть dynamic secrets, transit encryption, lease-based rotation. Клюши шифруются AES-256-GCM, в БД только ссылки (ref), никогда plaintext.

### 5. БД
- **PostgreSQL** — конфиг, audit log, метрики использования (тарификация)
- **Redis** — rate limiting, circuit breaker state, hot cache ответов
- **Опционально ClickHouse** — если хочешь аналитику по паттернам использования

**Минимальный MVP:** Go gateway + Redis + PostgreSQL + Vault wrapper. Остальное по мере роста.

## 🔨 Штрейкбрехер (разработчик)

### 1. Язык/фреймворк: Go
- Единый бинарник, нет зависимостей на рантайм
- Нативная конкурентность — критично для API-шлюза с множеством параллельных запросов
- Быстрый старт, низкий расход памяти
- FastAPI — хорош для MVP, но для продакшена с масштабом Go надёжнее
- Фреймворк: `gin` или `fiber` — оба проверены, fiber быстрее на бенчмарках

### 2. Rate limiting + Circuit breaker
- Rate limiter: токен-бакет или sliding window через Redis (`redis_rate` или `golang.org/x/time/rate`)
- Per-user, per-provider, per-endpoint — три уровня
- Circuit breaker: паттерн через `sony/gobreaker` или `hystrix-go`
- Пороги: 5 ошибок за 60 сек → open, 30 сек cooldown → half-open, успех → close
- Fallback: если провайдер A упал → маршрутизация на провайдер B с тем же интерфейсом

### 3. Конфигурация провайдеров: БД + YAML для дефолтов
- YAML — для статических названий, URL-ов, таймаутов, retry-политик
- PostgreSQL — для пользовательских ключей, квот, статистики
- Ключи шифровать через `crypto/aes` или vault
- Никогда не хранить ключи в коде или env

### 4. API Gateway: своё решение
- Kong/Traefik — оверкинд для начала
- Своё решение на Go — полный контроль над маршрутизацией, логированием, метриками
- Позже, при масштабировании — можно поставить Kong перед кластером
- Для MVP — один сервис на Go покрывает всё

### 5. Деплой: Docker Compose → Kubernetes
- Старт: Docker Compose (app + postgres + redis)
- Прод: Kubernetes с Helm-чартом
- Health checks, graceful shutdown, readiness/liveness probes
- CI/CD: GitHub Actions → build → test → push registry → deploy
- Мониторинг: Prometheus + Grafana (уже есть в инфраструктуре)

### Бонус — архитектура MVP:
```
Client → Go API Gateway → Provider Router → External APIs
                ↓
           PostgreSQL (keys, quotas)
           Redis (rate limits, cache)
           Prometheus (metrics)
```
Один сервис, три контейнера. Масштабируется горизонтально через K8s когда нагрузка вырастет.

## 🐱 Котолизатор (координатор)

### 1. Кто из агентов — основной пользователь
Все. Каждый агент использует LLM-запросы. Сейчас каждый тянет ключи из своего конфига — хаос. API Hub станет единым входом.

Приоритет по агентам:
- **owl, raven** — самые активные LLM-пользователи (аудит, исследования)
- **antcat** — интеграции с внешними API
- **streikbrecher** — кодогенерация
- **dominika** — разведка

### 2. Связь с существующими проектами
- **lab-vault** — хранилище ключей. API Hub должен брать ключи оттуда, а не дублировать. Vault — источник правды для секретов.
- **myrmex-control** — маршрутизация и мониторинг. API Hub должен писать метрики в myrmex (кто сколько тратит, rate limits).
- **free-api-hunter** — каталог бесплатных API. Можно интегрировать как источник fallback-провайдеров.

### 3. План запуска
- **MVP (1-2 недели):** FastAPI прокси с маршрутизацией по провайдерам (OpenRouter, OpenAI, Anthropic). Ключи из lab-vault. Единый `/v1/chat/completions` эндпоинт.
- **v2 (месяц):** Rate limiting, квоты по агентам, мониторинг через myrmex, fallback на free-api-hunter.
- **v3 (3 месяца):** Публичный продукт — документация, API keys для внешних пользователей, биллинг.

### 4. Риски на старте
- **Единая точка отказа** — если API Hub упадёт, все агенты теряют доступ к LLM. Нужен health-check и fallback на прямые ключи.
- **Безопасность ключей** — все ключи в одном месте. Если утечка — все провайдеры скомпрометированы. lab-vault обязателен.
- **Rate limits** — если один агент сожжёт квоту, пострадают все. Нужны per-agent квоты.
- **Зависимость от одного провайдера** — нужен fallback между OpenRouter → OpenAI → Anthropic автоматически.

**Рекомендация:** начать с MVP на FastAPI, интегрировать lab-vault сразу, мониторинг через myrmex. Не изобретать велосипед — использовать существующие проекты как основу.

## 🐦‍⬛ Ворон (исследователь)

### 1. Аналоги — кто уже делает подобное
- **OpenRouter** — хостед агрегатор 200+ моделей, единый API, бесплатные :free модели. Лидер рынка. На нём уже работает вся лаборатория.
- **LiteLLM** — open-source прокси, self-hosted. Можно поставить на свой сервер, маршрутизировать через десятки провайдеров. Полный контроль, но нужно поддерживать.
- **Merge Gateway** — коммерческий API-шлюз с мониторингом и аналитикой.
- **FreeLLMAPI** (10.5k ⭐ GitHub) — агрегатор бесплатных LLM API, 16 провайдеров, ~1.7B токенов/мес. Уже исследован лабораторией.
- **Zuplo** — edge-native API gateway, 300+ локаций, бесплатный tier, встроенный developer portal.
- **Kong / Tyk** — классические API gateways, но это infrastructure, а не агрегатор моделей.

### 2. Бесплатные API для интеграции
Приоритет для интеграции:
- **DaData** — адреса/компании/телефоны, есть бесплатный tier. Полезен для enrichment.
- **AbstractAPI** — валидация email/телефона/IP, бесплатный tier. Простая интеграция.
- **ScraperAPI** — обход блокировок при скрапинге, бесплатный tier 1000 запросов/мес. Критично для разведки.
- **PDFGeneratorAPI** — генерация PDF, бесплатный tier. Полезен для отчётов.
- **Clearbit** — enrichment компаний/людей, но дорогой. Лучше заменить на AbstractAPI.

Из проекта Free API Hunter — 16 бесплатных LLM провайдеров (Google, Groq, Mistral, OpenRouter, Together AI, Kilo, Pollinations, LLM7, OVH).

### 3. Подводные камни
- **Rate limits** — у каждого провайдера свои лимиты. Нужен circuit breaker и очередь запросов. У OpenRouter free — 20 req/min, 200 req/day.
- **Блокировки** — Google блокирует consumer API для коммерческого использования. NVIDIA — только для evaluation. Нужна ротация ключей.
- **Юридические вопросы** — нельзя перепродавать чужие API-ключи (нарушение ToS). Нужна модель "пользователь загружает СВОЙ ключ". Мы не храним чужие токены.
- **Задержка** — каскадный fallback увеличивает latency. Нужен smart routing.
- **Стоимость** — бесплатные лимиты быстро исчерпываются при 131к запросов/мес. Нужен pay-per-use fallback.
