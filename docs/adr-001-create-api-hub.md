# ADR-001: Создание API Hub — единого API-шлюза

**Статус:** принято
**Дата:** 2026-06-19
**Автор:** ЗавЛаб
**Консультанты:** Мангуст, Штрейкбрехер, Котолизатор, Ворон, Доминика

---

## Контекст

Колония из 8 агентов использует множество внешних API через разные провайдеров:
- LLM: OpenRouter, OpenAI, Anthropic, Google, Groq, Mistral, Together AI, Kilo, Pollinations, LLM7, OVH (16+ бесплатных)
- Гео/адреса: DaData
- Валидация: AbstractAPI
- Скрапинг: ScraperAPI
- Генерация: PDFGeneratorAPI

**Проблемы текущего состояния:**
- Каждый агент хранит ключи в своём конфиге — хаос
- Нет единой точки мониторинга расходов
- Нет fallback при падении провайдера
- Нет rate limiting — один агент может сжечь квоту для всех
- Нет аналитики — кто, сколько, куда расходует

**Существующие решения (OpenRouter, LiteLLM, Merge Gateway) не подходят:**
- Все заточены только под LLM-провайдеров
- Нет универсального шлюза для любых API (гео, скрапинг, генерация документов)
- OpenRouter не поддерживает BYOK для не-LLM API

---

## Решение

Создать **API Hub** — единый API-шлюз для всех внешних API.

**Ключевые принципы:**
- BYOK (Bring Your Own Key) — пользователь загружает свои ключи
- Единый эндпоинт — один интерфейс для всех провайдеров
- Универсальность — не только LLM, а любые API
- Масштабируемость — заложена с первого дня

---

## Последствия

### Положительные
- Единая точка входа для всех внешних API колонии
- Полный контроль расходов и квот
- Fallback между провайдерами автоматически
- Безопасное хранение ключей через lab-vault + HashiCorp Vault
- Аналитика использования

### Отрицательные
- Единая точка отказа — нужен health-check и fallback на прямые ключи
- Усложнение инфраструктуры — новый сервис нужно поддерживать
- Безопасность — все ключи в одном месте требует усиленной защиты

### Риски и митигация
| Риск | Митигация |
|---|---|
| Единая точка отказа | Health-check + fallback на прямые ключи |
| Утечка ключей | HashiCorp Vault + AES-256-GCM |
| Сжигание квот одним агентом | Per-agent квоты |
| Зависимость от провайдера | Fallback цепочка + free-api-hunter |
